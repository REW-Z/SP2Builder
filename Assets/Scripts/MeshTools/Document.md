# Cut 的逻辑  

1. 给定一个平面 Plane。
2. 遍历 Mesh 的每个三角形。
3. 判断三角形三个顶点在平面的哪一侧：
4. 全在正侧：放入 positive mesh
5. 全在负侧：放入 negative mesh
6. 跨过平面：把三角形切开
7. 如果 cap = true，把切口边收集起来，组成闭合环。
8. 对闭合环做三角化，补上切口盖子。  

# Subtract 的逻辑  

Subtract(A, B) 做的是：

1. 给 A 建 BSP。
2. 给 B 建 BSP。
3. 用 B 的空间划分去裁剪 A。
4. 用 A 的空间划分去裁剪 B。
5. 删除 A 中落在 B 内部的面。
6. 保留 B 中落在 A 内部的面，并翻面作为洞壁。
7. 合并结果。

用另一个 BSP 树裁掉“在对方实体内部”的面。  
（a.ClipTo(b)意思是用 b 这棵 BSP 树去裁剪 a 里面的多边形。）

Build：  
把保留下来的面重新合进 BSP 树。  

# 补充 1  

“用 B 的空间划分去裁剪 A”意思是：

**先用 B 的所有三角面建立一棵 BSP 树，这棵树能判断任意面片在 B 的外部、内部，还是跨过 B 的边界。然后把 A 的面片丢进这棵树里，让 B 的 BSP 把 A 的面片切开并删除落在 B 内部的部分。**

举个例子，`A` 是方块，`B` 是球。

A 的一个三角面可能穿过球体：

```text
A 的一个面片
一部分在球外
一部分在球内
```

用 B 的 BSP 裁剪时，会发生：

```text
A 面片进入 B 的 BSP
-> 被 B 的某些分割平面切开
-> 分成若干小面片
-> 在 B 外部的小面片保留
-> 在 B 内部的小面片删除
```

所以结果就是：

```text
A 被 B 占据的那部分挖掉
```

这里的“B 的空间划分”不是只看 B 的一个三角面，而是 B 的整棵 BSP 树。它相当于用 B 的表面把空间分成：

```text
B 外部
B 内部
B 边界
```

然后用这个划分去处理 A 的三角形。
